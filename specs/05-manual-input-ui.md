# Spec 05 — 수동 입력 화면 (ManualInputPage)

## 파일 위치

`src/pages/ManualInputPage.tsx`

## 라우트

`/manual`

## 목적

JSON 파일 없이 1건씩 직접 입력해서 저장.
기존 saju-cube PersonForm.tsx + DateInputPage.tsx 기능을 이 프로젝트에 맞게 재구현.

## 화면 구성

```
┌────────────────────────────────────────────────────┐
│  ← 뒤로   수동 입력                                 │
│                                                    │
│  이름 *        [________________]                  │
│  성별 *        ( 남성  ○ 여성 )                    │
│  생년월일 *    [연도] [월] [일]   ○양력 ○음력       │
│  생년월시      [시]   [분]        □ 시간 모름        │
│  메모          [________________________________]   │
│                                                    │
│  ── 6개 명식 미리보기 ────────────────────────────  │
│  [계산하기] 버튼 클릭 시 아래에 6개 날짜 표시        │
│  본원-2  본원-1  본원   본원+1  본원+2  본원+3      │
│  YYYY.MM.DD ...                                    │
│                                                    │
│                          [저장]                    │
└────────────────────────────────────────────────────┘
```

## 필드

| 필드 | 필수 | 입력 방식 |
|---|---|---|
| 이름 | ✅ | text input |
| 성별 | ✅ | radio (남성/여성) |
| 생년월일 | ✅ | 숫자 3개 입력 (year/month/day) |
| 음양력 | ✅ | radio (양력/음력) |
| 생시 (hour) | ❌ | select (자시~해시 12개 + 시간 모름) |
| 분 (minute) | ❌ | number input (0~59) |
| 메모 | ❌ | textarea |

## 시간 옵션 (시간 모름 선택 시 hour=11, minute=0 기본값)

| 시신 | 시간 범위 | hour 값 |
|---|---|---|
| 자시 | 23:00 ~ 01:00 | 0 |
| 축시 | 01:00 ~ 03:00 | 2 |
| 인시 | 03:00 ~ 05:00 | 4 |
| 묘시 | 05:00 ~ 07:00 | 6 |
| 진시 | 07:00 ~ 09:00 | 8 |
| 사시 | 09:00 ~ 11:00 | 10 |
| 오시 | 11:00 ~ 13:00 | 12 |
| 미시 | 13:00 ~ 15:00 | 14 |
| 신시 | 15:00 ~ 17:00 | 16 |
| 유시 | 17:00 ~ 19:00 | 18 |
| 술시 | 19:00 ~ 21:00 | 20 |
| 해시 | 21:00 ~ 23:00 | 22 |

## 6개 명식 미리보기

"계산하기" 버튼 클릭 시:
1. `calculateAutoDates(year, month, day)` 호출
2. 실패 시 에러 메시지 표시
3. 성공 시 6개 날짜 카드로 표시

## 저장 동작

- `convertRecord()` 함수로 `MinimalPersonData` 생성
- Supabase `POST /persons` 호출
- 성공: "저장되었습니다" 토스트 + 폼 초기화
- 실패: 에러 메시지 표시

## 유효성 검사

- 이름 비어있으면 저장 버튼 비활성화
- 연도: 1600~2100
- 월: 1~12
- 일: 1~31 (기본 검증, 실제 유효성은 계산 시 체크)
- 음력 입력 시 `lunartosolar()` 결과로 날짜 유효성 확인
